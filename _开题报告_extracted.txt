杭州电子科技大学
毕业设计（论文）开题报告
综述本课题国内外研究动态，说明选题的依据和意义
1.1研究背景与现状
近年来，随着深度学习技术的突破，尤其是大型语言模型（Large Language Model, LLM）的迅猛发展，人机交互、知识服务与智能决策领域迎来了范式转移。自OpenAI发布GPT系列模型以来，包括ChatGPT、Claude、Gemini及Copilot在内的生成式人工智能产品，展现出卓越的自然语言理解与代码生成能力。AI技术正加速从科研实验室走向社会应用，特别是在教育领域，智能化辅助工具已成为推动个性化学习与教育资源均衡化的重要力量。传统的在线教育平台（如Coursera、学堂在线）虽然引入了基础的算法推荐，但往往局限于标准化的知识传递，难以提供深度的交互式辅导与针对性的即时反馈。
在计算机科学与技术及相关专业中，C/C++语言具有不可替代的基础地位。作为操作系统、嵌入式系统及高性能计算的核心语言，C/C++不仅要求学习者掌握复杂的语法结构，更需要深刻理解指针逻辑、内存管理及底层计算机体系结构。然而，这种“高门槛、强逻辑”的特性使得初学者往往面临极高的认知负荷，常因“指针概念抽象”、“内存错误难以调试”而陷入学习困境。当前，虽然通用大语言模型能够辅助代码编写，但在C/C++教学场景中仍存在显著局限：“幻觉”风险与规范性缺失：C/C++对内存操作的安全性要求极高。通用大模型生成的代码常出现引用非标准库、内存泄漏或未定义行为（Undefined Behavior），这对缺乏辨别能力的初学者构成了严重的误导风险。缺乏垂直领域的深度交互：通用模型往往给出直接答案，缺乏引导式的调试思路与底层原理剖析，难以真正培养学生的工程思维与问题解决能力。
另一方面，在就业导向的职业发展环节，计算机专业学生正面临日益严峻的挑战。随着互联网与硬科技产业的升级，企业对研发人员的筛选标准从单纯的“代码实现”转向了“底层原理掌握”与“工程素养”的双重考核。应届毕业生普遍存在“理论与实战脱节”的问题，具体表现为简历缺乏技术深度、难以应对企业级C/C++笔试题、以及在技术面试中对底层机制（如内存模型、编译原理）阐述不清。现有的就业指导模式主要依赖传统的讲座与人工咨询，存在资源有限、覆盖面窄、个性化不足等问题。为解决这一问题，模拟面试在就业指导课中被广泛使用[1][2]。但针对C/C++这类强技术学科，仅靠临近毕业时的突击指导显然不足，专业素养的培养不能仅依赖临近毕业时的就业指导课[3]，学生迫切需要一个能够贯穿日常学习到求职全周期的智能化系统，通过常态化的代码训练、规范的纠错反馈以及沉浸式的模拟面试，来系统性地积累专业素养与面试经验。
综上所述，构建一个聚焦C/C++垂直领域，集成多智能体协同（Multi-Agent System）技术的智能助教系统具有重要的现实意义。该系统旨在超越通用模型的泛化能力，通过构建专精于“代码规范审查”、“底层原理答疑”及“技术面试模拟”的不同智能体角色，实现从基础学习纠错到高端职业辅导的全流程覆盖。这不仅能有效规避通用模型在专业编程教学中的“幻觉”风险，更能契合计算机专业大学生高质量就业与个性化成长的迫切需求。
1.2国外研究与应用进展
国外在人机交互与智能系统的研究起步较早。作为人机交互的核心技术之一，对话系统用来实现人机对话，提供特定的自动化服务，使人与机器之间可以像人与人之间一样使用自然语言沟通[4]。对话系统的发展过程可被分为三个阶段。上世纪60年代，被视为心理医生对患者进行干预的Eliza的问世，Eliza被认为是第一个对话系统[5]。第一代对话系统是通过匹配关键字和回复规则来实现人机对话[6]，其优点是易于理解、简单，缺点是规则的构建较复杂，扩展性不强，并且需要大量的物力人力[7]。第二代对话系统基于数据驱动，只进行浅层学习，需要特征转换或者人工提取，其缺点是难以扩展[8]。第三代对话系统（即当前大型语言模型的前身）将深度学习作为主要方法，此类方法模型的优点是表征能力较强，缺点是需要大规模的数据，随着数据的不断增长，此类方法具有广阔的发展前景。在比较经典的对话系统架构中，通常将整个系统划分为四个部分，分别是自然语言理解（Natural Language Understanding, NLU）、对话状态跟踪（Dialogue State Tracking, DST）、对话策略学习（Policy Learning, PL）和对话生成（Natural Language generation, NLG）[9]。
然而在具体的编程教育应用层面，目前的解决方案仍存在显著局限。传统的代码评测系统（Online Judge, OJ）如LeetCode、Codecademy等平台提供了丰富的题库与即时编译环境，但其反馈机制主要基于测试用例的通过率，缺乏对C/C++代码中特有的内存管理、指针逻辑错误（如野指针、内存泄漏）的深层语义分析，无法为初学者提供“为什么错”的解释。另一方面，GitHub Copilot、Cursor等新兴AI工具虽然在代码补全上表现出色，但其设计初衷是服务于成熟开发者提高效率，直接给出最终代码而非引导思考，容易导致学生产生依赖心理。特别是针对C/C++这一垂直领域，通用的AI模型往往无法严格区分“C语言的底层过程式特性”与“C++的高级面向对象特性”，常在解释指针操作或虚函数模型时出现概念混淆或风格杂糅。 此外，针对计算机专业求职的“AI模拟面试”类产品目前尚处于起步阶段，大多仅能进行简单的行为面试（Behavioral Question），缺乏对C/C++复杂的内存模型、并发控制、STL原理等硬核技术问题的深度多轮交互能力。
这些研究与应用现状表明，虽然AI已经能够在“人机共创”场景中发挥作用，但如何构建一个专注于系统编程领域（System Programming），让多个AI智能体协同决策（如一个Agent负责代码规范纠错，另一个Agent负责模拟大厂技术面试），仍是当前的研究难点。
1.3国内研究与应用进展
国内高校与企业近年来也在积极探索AI与教育的结合。教育智能化方向：清华大学推出的“雨课堂”、北京大学的“学堂在线”平台均引入AI辅助教学模块；百度、阿里、华为等企业相继推出文心一言、通义千问、盘古等大模型，具备强大的中文理解与问答能力。随着这些大型语言模型的普及，如何高效、安全地使用它们成为新的研究热点。特别是对于语法严谨的C/C++语言教学，如何通过精准的指令控制模型的输出风格至关重要。
提示工程（Prompt Engineering, PE）作为关键技术应运而生。Prompt工程被定义为是将AI提示进行定制以达成特定目标的技能[10]，也是一门为人工智能模型制作有效提示的艺术。它已成为决定人工智能（人工智能）生成输出质量和有用性的关键因素，这种实践涉及战略性地设计和结构提示，以引导人工智能模型实现预期的结果，确保它们生成相关、信息丰富且准确的响应[11]。研究者总结了Prompt的设计原则，如：给模型清晰具体的指令、给模型充足的思考时间、将问题从难到易进行分解、优先采用渐进式提示等[12]。之所以需要专门的提示工程，是因为尽管ChatGPT等模型的回答具有可控性，但一个不完善的Prompt则可能会使其不能完全发挥潜力。许多用户发现其回答通常会出现概念界定模糊、知识内容表层化、推理能力不足、语义理解偏差、机械式应答、缺乏人性化特征等问题[13]。
此外，大型语言模型面临的另一大挑战是“幻觉”问题，即大模型生成与现实之间界限模糊的、具有欺骗性的虚假内容，包括但不限于常识、编程、归因错误以及更高层次的概念误导[14]。这一问题在C/C++编程辅助中尤为致命，例如模型可能会捏造不存在的库函数或错误的内存释放方式，导致程序崩溃。针对大模型幻觉修正层面，已有研究者如Peng等引入外部知识库，通过检索、生成回复和评估进行反复迭代来减少模型在回答过程中由于知识不足而产生的幻觉[15]；针对低资源少样本的大模型幻觉情况的研究也正在开展，这也为本课题构建基于检索增强生成（RAG）的C/C++专属知识库提供了理论支撑。在职业服务领域，Boss直聘、智联招聘等平台虽提供岗位推荐，但其智能化程度有限，缺乏可交互式的技术指导。现有的系统难以模拟计算机专业校招中高强度的C/C++技术考察（如指针深度理解、算法手撕、底层原理追问）。
由此可见，构建一个基于多智能体协同、集成RAG技术以抑制幻觉、并专注于C/C++深度学习与技术面试辅导的系统， 尚处于探索期，具备较高的创新价值与应用潜力。
1.4选题依据与研究意义
从国内外研究现状来看，虽然人工智能技术正在重塑教育形态，但现有的智能辅助工具在专业性与精准度上仍存在显著错位。对于计算机专业的核心基础——C/C++语言而言，其涉及的内存管理、指针操作、并发控制等底层机制极为复杂。目前的通用大模型（如ChatGPT）在处理此类严谨逻辑时，常因缺乏领域约束而产生代码“幻觉”（如引用不存在的库、错误的内存释放），无法满足高质量教学的需求。此外，现有的职业辅导平台多集中于简历优化或岗位推荐等浅层服务，难以提供针对C/C++开发岗（如后台开发、嵌入式系统）所需的深度技术面试模拟与算法逻辑训练。
国外虽有Cognitive Tutor等智能导学系统，但多基于规则匹配，缺乏生成式AI的灵活性；国内研究则多侧重于通用教学辅助，尚未形成“学-练-测-面”一体化的垂直领域解决方案。因此，构建一个集成检索增强生成（RAG）技术、支持多智能体协同（如代码审查员、技术面试官、学习规划师协作）的C/C++智能辅助系统，不仅能有效抑制模型幻觉，更能精准回应计算机专业学生攻克底层技术难点与应对高难度技术校招的现实需求。
本研究的开展具有多方面的重要意义：理论上本项目将探索大型语言模型（LLM）与多智能体系统（Multi-Agent System）在垂直编程领域的融合应用模式。特别是通过引入外部知识库（RAG）来增强模型在特定编程语言（C/C++）教学中的事实准确性与逻辑严密性，有助于丰富智能教育中关于“领域知识注入”与“复杂逻辑推理”的理论基础。实践中项目将实现一个专注于C/C++领域的智能辅导平台，解决传统教学中代码反馈滞后、实战演练不足的痛点。通过智能体扮演“技术面试官”和“代码Reviewer”，系统能帮助学生从单纯的语法记忆进阶到工程实践，切实提升其代码质量、算法思维及应对互联网大厂技术面试的核心竞争力。社会方面上 C/C++是构建数字基础设施（操作系统、高性能计算）的基石。本研究成果有助于提升高校计算机专业人才的底层技术素养，缓解高校人才培养与企业高阶技术需求之间的脱节问题，从而推动高质量就业与教育资源的数字化转型。
二、研究的基本内容，拟解决的主要问题：
2.1研究目标
本课题的总体目标是开发一个基于多智能体协同与RAG（检索增强生成）技术的C/C++垂直领域辅助学习与面试系统。该系统旨在为计算机专业学生提供从“基础语法攻关”、“复杂工程实践”到“大厂技术面试”的一站式解决方案。一方面是构建深度垂直的“AI代码导师”： 区别于通用的聊天机器人，本系统将挂载C/C++标准文档与经典技术书籍（如《C++ Primer》、《Effective C++》）作为私有知识库，确保对指针、内存管理、STL容器等底层概念的解释准确无误，解决通用大模型在编程教学中的“幻觉”问题。另一方面是实现“学-练-测-面”全流程闭环： 通过多个具备专业分工的智能体（Agent），不仅能为学生规划从C语言入门到C++高阶开发的学习路径，还能对学生提交的代码进行深度Review（审查），并基于代码逻辑直接发起模拟技术面试，切实提升学生的工程代码能力与就业竞争力。
2.2系统主要功能
2.2.1 用户注册与个性化画像模块
系统要求用户注册登录。该模块负责建立用户档案，记录其 C/C++ 学习进度与代码提交历史。基于这些数据，系统利用 Coze 的长期记忆功能，为每位用户构建动态的“技术能力画像”，确保 Agent 在后续交互中能够调用用户的历史上下文，提供个性化服务。
2.2.2 C/C++垂直领域多智能体矩阵
基于 Coze平台 构建一组专注于系统编程领域的智能体，各司其职并高效协作：C/C++学习路径规划师责评估用户的编程基础，制定如“30天精通C++ STL”或“嵌入式C语言进阶”的个性化学习路线；智能代码审查员区别于普通编译器报错，该角色专注于代码逻辑分析。它可以检测内存泄漏（Memory Leak）、野指针风险、非规范命名以及代码风格问题，并给出具体的重构建议；技术面试官是模拟互联网大厂后端/嵌入式岗位的面试风格。它会根据用户的代码水平，追问底层原理（如“vector扩容机制”、“虚函数表结构”），并支持多轮深挖追问；知识库问答助手可以集成RAG技术，基于权威文档回答语法疑难，避免胡编乱造。
2.2.3专属知识库构建与模型调优训练
为保障AI输出的专业性并抑制大模型在C/C++领域的“幻觉”，本系统专门设计了专属知识库与模型调优模块作为核心的“训练”环节。系统在后台构建了包含ANSI C标准、ISO C++标准、Linux系统编程手册及LeetCode高频题解在内的向量数据库，作为系统的核心“大脑”。在此基础上，系统通过调整文本分块与检索阈值优化召回准确率，并结合Prompt Engineering进行少样本（Few-shot）引导训练，向模型注入典型的C/C++错误代码负样本与标准修改正样本，规范其纠错逻辑与输出格式。当用户提问或上传代码时，系统优先检索知识库中的权威内容，结合调优后的提示词生成回答，确保技术解释的绝对严谨。
2.2.4 自动化评估与系统边界测试
为确保系统的稳定性与指导建议的准确性，在应用落地前引入了严格的自动化评估与测试机制。该模块利用平台自带的调试环境，针对每个核心智能体构建了包含极端的指针越界、并发死锁等边界情况的测试数据集，以此验证代码审查员和知识库助手等角色是否能准确识别错误并给出合理建议。同时，系统在内测运行阶段会收集测试用户的反馈数据，评估各个智能体在处理高并发死循环、超深递归等恶意代码输入时的表现，并据此持续迭代优化提示词策略与知识库检索权重，完成系统的闭环测试。
2.2.5 AI协同技术面试工作流
该模块将前期构建与调优的底层能力转化为具体的业务流应用。在代码演练场景中，用户输入代码后不仅能触发沙箱环境执行编译测试，系统还会调用代码审查员进行静态语义分析，例如在链表操作中自动提示成环的边界情况并推送相关知识点。在求职实战场景中，系统提供AI协同面试工作流，首先解析用户简历中的特定技术栈并生成针对性考题；用户提交解题代码后，审查员评估代码优缺点，随后技术面试官基于代码缺陷发起如“锁机制性能影响”等真实大厂视角的压力追问，真正实现从理论学习到求职实战的无缝衔接。
2.2.6 程序员能力雷达图模块
该模块通过整合用户在整个系统中的学习与交互数据，将隐性的学习成果转化为可视化的六维能力模型。系统通过算法动态分析用户的测验与面试表现，生成涵盖基础语法、内存管理、数据结构与算法、面向对象设计、STL/库使用、系统编程能力六个维度的能力雷达图。这不仅能帮助学生精准识别如“算法强但内存管理弱”的技术短板，该数据指标还将反向反馈给学习路径规划师智能体，指导其动态调整后续推送的课程内容与实战练习难度，实现个性化学习的闭环应用。
2.3拟解决的关键问题
2.3.1 编程领域大模型的“幻觉”抑制与知识准确性
通用大模型容易生成这就“C++风格的C代码”或引用不存在的库函数。关键在于如何通过Prompt Engineering与RAG技术，强制模型严格遵循特定的语言标准（如C99或C++11），并限制其仅基于知识库内容回答，确保教学内容的准确性。
2.3.2代码逻辑的深度语义理解
如何让AI不仅仅是充当“语法检查器”，而是能像资深工程师一样进行Code Review。我们需要设计复杂的CoT（思维链）提示词，引导模型逐步分析代码的时间复杂度、空间复杂度及潜在的边界条件错误。
2.3.3多智能体上下文共享与角色切换
在模拟面试中，如何保证“面试官”能读取“审查员”对代码的评价，从而提出有针对性的追问，而不是机械地换题。这涉及到Agent之间的Memory共享机制的设计，确保对话逻辑的连贯性与压迫感。
2.3.4个性化题目推荐算法
系统会基于用户的“能力雷达图”动态调整题目难度。例如，当检测到用户在“指针”模块频繁出错时，系统应自动增加相关练习的权重，而非机械推送下一章节。
三、研究步骤、方法及措施：
3.1研究步骤
3.1.1 需求分析阶段
重点调研计算机专业学生在 C/C++ 学习全周期中的具体痛点（如：指针与内存管理的理解障碍、STL 底层原理的认知缺失、校招技术面试的实战经验不足）。同时，分析互联网大厂对 C/C++ 岗位的技能要求，明确系统应具备的核心能力（如代码深度纠错、并发编程考核、底层原理问答）。在此基础上，定义系统的非功能性指标，特别是对 AI 生成代码准确性与响应速度的要求。
3.1.2系统设计阶段
绘制前后端交互图，重点设计用户代码提交、沙箱执行与 Agent 反馈的数据流转路径。规划向量数据库（Vector DB）的结构，确定需要导入的 C/C++ 权威文档（如 ISO 标准、Linux Man Pages、经典书籍），并设计相应的索引策略以优化 RAG 检索效果。智能体编排，绘制“代码审查员”、“技术面试官”等核心 Agent 的状态机流转图，明确它们在不同教学场景下的协作逻辑。
3.1.3核心模块实现阶段
前端采用 React + Vite + Monaco Editor（代码编辑器组件）搭建沉浸式编程
界面；后端使用 Next.js 构建业务逻辑，集成 Docker 容器技术作为代码安全执行沙箱。通过 API 接入大语言模型，并利用 Coze 平台搭建 Agent 后端服务，实现从“用户输入代码”到“AI 生成分析报告”的完整链路。重点在于确保系统能准确解析复杂的 C++ 语法结构。
3.1.4智能协同与算法实现阶段
我们计划基于 Coze 平台 的多智能体（Multi-Agent）机制，构建不同角色的 AI 助手，并通过配置化的方式实现智能协同。在这一阶段，我们并不是从零开始搭建复杂的智能体调度算法，而是借助 Coze 提供的 可视化工作流与自定义 Agent 配置能力，对各个角色型智能体进行独立设计与优化。每个 Agent 都有清晰的职责与上下文语境，比如知识库挂载，为“教学助手”Agent 配置 RAG 检索策略，确保其回答基于权威文档，严控幻觉。并且实现Prompt 深度调优，针对“代码审查员”，设计包含思维链的 Prompt，使其能输出包含“错误定位—原因分析—修改示例”的结构化反馈；针对“面试官”，配置追问逻辑，使其能根据用户的代码漏洞进行深度盘问。
3.1.5系统测试与优化阶段
除了常规的功能测试外，重点进行“领域准确性测试”：邀请 C/C++ 成绩优秀的学生或老师对 AI 生成的解释、代码修正建议进行准确性评估。根据反馈，迭代优化 Prompt 模板和知识库的召回策略，最大限度降低错误率。同时，测试代码沙箱在高并发下的安全性与稳定性。
3.1.6总结与论文撰写阶段
汇总实验数据，撰写系统实现报告与论文。
3.2研究方法
3.2.1 文献研究与知识工程方法
不仅仅是查阅论文，重点在于垂直领域知识库的构建。我们会收集整理《C Primer Plus》、《Effective C++》、C99/C11 标准文档等高质量语料，采用文本分块与向量化技术，构建高质量的 C/C++ 领域知识图谱，为 AI 提供准确的“外挂大脑”。
3.2.2多智能体协同建模方法
参考“观察—思考—行动”（ReAct）范式，设计 Agent 之间的交互协议。例如，当“代码审查员”发现严重内存泄漏时，自动触发“知识讲解员”介入，推送相关的内存管理课程，实现角色间的无缝配合。
3.2.3提示词工程优化方法
通过不断优化Prompt模板与上下文管理机制，提升AI输出结果的准确性和稳定性，确保系统能提供贴合用户实际的建议与内容生成。
3.2.4数据可视化方法
使用 ECharts 生成“C/C++ 技能六维雷达图”（涵盖语法、算法、内存、指针、面向对象、系统编程），基于用户的做题与面试数据实时更新，直观展示技术短板。
3.3实施保障措施
项目在技术实现上，将依托 Coze 智能体编排平台 与 主流大语言模型（如豆包）作为底层支撑，通过复用成熟的 API 接口降低底层算法的开发复杂度，从而使研发重心聚焦于 Prompt 提示词工程 的优化与 C/C++ 垂直业务逻辑的实现；前端则集成 Monaco Editor 等成熟编辑器组件，以保障流畅的编码体验。针对大模型可能产生的幻觉问题，项目拟采取 “源头控制 + 运行时校验” 的双重策略：在知识库构建初期，通过人工筛选确保入库文档（如标准手册、经典教材）的权威性；在系统运行期间，设置回答置信度阈值，对 AI 信心不足的生成内容进行标记或拦截。此外，考虑到 C/C++ 代码运行的高风险性，系统将引入 Docker 容器隔离技术 构建代码沙箱，严格限制用户提交代码的系统调用权限与资源占用，防止恶意代码破坏服务器环境，配合全链路的数据加密传输，切实保障系统的安全稳定运行。
